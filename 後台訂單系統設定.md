# 後台訂單系統設定

這個網站是放在 GitHub Pages，上面不能直接儲存訂單。建議使用 Google Sheets 當訂單後台，客人送出的表單會自動寫入試算表。

## 訂單欄位

Google Sheets 第一列請建立以下欄位：

- 建立時間
- 商品名稱
- 姓名
- 聯絡電話
- 數量
- 需要日期
- 備註需求
- 狀態

狀態可以手動填：

- 新訂單
- 已聯絡
- 製作中
- 已完成
- 已取消

## Google Apps Script 程式

在 Google Sheets 裡點：

```txt
擴充功能 → Apps Script
```

貼上 `orders-apps-script.gs` 的內容，部署成 Web App 後，將取得的 Web App URL 貼到 `config.js`：

```js
window.ORDER_API_URL = "你的 Google Apps Script Web App URL";
```

設定完成後，客人在網站送出下單詢問，就會寫入 Google Sheets。

## 後台管理方式

直接打開 Google Sheets 管理訂單即可。這比把後台放在公開 GitHub Pages 安全，因為客人資料不會公開在網站裡。
